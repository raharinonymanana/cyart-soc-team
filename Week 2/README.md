# Week 2

This folder contains the complete Week 2 submission for the SOC practical and theoretical task.

## Objectives
The purpose of this week’s work is to understand how a SOC analyst:
- assigns alert priority
- classifies security incidents
- performs basic incident response
- documents investigation activity
- preserves evidence
- reports a complete alert-to-response cycle

## Folder Structure

### 01_Theory
Contains the theoretical study material for:
- Alert Priority Levels
- Incident Classification
- Basic Incident Response

### 02_Alert_Management
Contains practical alert management work, including:
- alert classification table
- priority scoring
- Wazuh dashboard evidence
- incident ticket draft
- escalation email

### 03_Response_Documentation
Contains documentation artifacts, including:
- incident response template
- investigation timeline
- phishing checklist
- post-mortem summary

### 04_Alert_Triage
Contains triage practice documents and IOC validation results.

### 05_Evidence_Preservation
Contains evidence preservation records such as memory dump references, hashes, and chain-of-custody style documentation.

### 06_Capstone
Contains the final end-to-end incident scenario from attack simulation to response and reporting.

### 07_Screenshots
Contains screenshots supporting the practical work.

### 08_Workflow
Contains the step-by-step explanation of how the task was completed.

## Summary
This Week 2 folder is organized to clearly separate theory, practical exercises, screenshots, and workflow documentation.
