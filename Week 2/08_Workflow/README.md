# Workflow Steps

## Step 1: Studied theoretical concepts
Reviewed alert priority levels, incident classification methods, and the incident response lifecycle to build the foundation for the practical tasks.

## Step 2: Prepared alert management artifacts
Created an alert classification table, assigned severity levels, mapped alerts to MITRE ATT&CK where applicable, and prepared a sample incident ticket and escalation email.

## Step 3: Documented response procedures
Prepared incident response documentation including an incident template, a timeline of actions, a phishing investigation checklist, and a short post-mortem summary.

## Step 4: Performed alert triage practice
Reviewed sample or simulated alerts, assigned priority, recorded status, and validated indicators of compromise using threat intelligence references where possible.

## Step 5: Documented evidence preservation
Prepared evidence collection records, hash references, and chain-of-custody style documentation to demonstrate correct evidence handling practices.

## Step 6: Completed capstone exercise
Documented the full alert-to-response cycle, including attack simulation context, detection, triage, response actions, and reporting.

## Step 7: Collected screenshots and supporting files
Saved screenshots from dashboards, alerts, tickets, and workflow evidence to support the written documentation.

## Step 8: Organized the GitHub repository
Uploaded all final documents into clearly named folders under Week 2 and added README files for clarity and structure.
